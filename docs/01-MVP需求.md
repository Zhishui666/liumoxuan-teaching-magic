# MVP 需求

## MVP 目标

跑通“一份核心教学内容 -> 多个场景版本”的最小闭环，先证明减负增效成立，再决定是否开发界面或自动化脚本。

## 范围内

- 结构化输入模板。
- 版本适配矩阵。
- 一份核心 Prompt 合约。
- 至少一个真实教学样例。
- 至少三种输出版本。
- 人工验收记录：可用点、需改点、下一轮优化点。

## 范围外

- 暂不做完整 Web 应用。
- 暂不接入学生数据库。
- 暂不自动生成 PDF、PPT 或飞书文档。
- 暂不替代教师对知识准确性的最终审核。

## 用户流程

1. 教师粘贴一份核心教学内容。
2. 教师选择或填写目标输出场景。
3. 适配器识别核心不变量：知识点、目标、例题、易错点、任务。
4. 适配器按矩阵生成多个版本。
5. 教师审核并记录修改点。

## 最小验收样例

建议第一份样例使用语文学科中的一个真实知识点，例如：

- 文言实词一词多义
- 现代文阅读中的“内容概括”
- 作文素材如何从一次经历迁移到多个题目
- 诗歌意象不能死记固定含义

## 下一步迭代

- v0：Markdown + Prompt 手动跑通。
- v1：命令行脚本读取输入模板并生成输出文件。
- v2：Web 表单选择场景矩阵。
- v3：接入课堂反馈、讲义和家长沟通的真实工作流。
