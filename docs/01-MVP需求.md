# MVP 需求

## MVP 目标

跑通“知识点资产 + 学生/班级学情 -> 多个教学出口”的最小闭环，先证明减负增效成立，再决定是否扩展为完整 Web 工具。

## 范围内

- 知识点卡片模板。
- Markdown 编辑器或 Markdown 文件输入。
- 学生/班级学情输入。
- 版本适配矩阵。
- 一份核心 Prompt 合约。
- 至少一个真实教学样例。
- 至少三种输出版本。
- 人工验收记录：可用点、需改点、下一轮优化点。

## 范围外

- 暂不做完整多用户 Web 应用。
- 暂不接入真实学生数据库。
- 暂不做自动音频转写，先支持粘贴转写文本。
- 暂不自动生成 PDF、PPT 或飞书文档。
- 暂不替代教师对知识准确性的最终审核。

## 用户流程

1. 教师选择或新建一个知识点。
2. 教师补充讲法、注意点、误区、教材和题目资源。
3. 教师选择学生或班级，并粘贴学情/课堂转写。
4. 适配器识别核心不变量：知识点、目标、例题、易错点、任务。
5. 适配器按矩阵生成多个版本。
6. 教师审核并记录修改点。
7. 优质输出保存为复用案例。

## 最小验收样例

建议第一份样例使用语文学科中的一个真实知识点，例如：

- 文言实词一词多义
- 现代文阅读中的“内容概括”
- 作文素材如何从一次经历迁移到多个题目
- 诗歌意象不能死记固定含义

## 下一步迭代

- v0：Markdown + Prompt 手动跑通。
- v1：单页 Web 工作台，支持知识点编辑、学情粘贴、结构化输出预览。
- v2：本地文件数据库，保存知识点、学生画像、生成任务和复用案例。
- v3：接入课堂反馈、讲义和家长沟通的真实工作流。
- v4：接入 STT、题库自动检索和 PDF 下载。
